# Campus Field Finder – Project Documentation

## Overview
Campus Field Finder is a modern web application that enables users to post and discover campus field opportunities. The frontend is built with **React (Next.js) + TypeScript**, styled using **Tailwind‑like custom CSS** with a dark‑mode, glass‑morphism aesthetic.

## Repository Structure
```
frontend/
├─ app/                # Next.js app routes (dashboard, admin)
├─ components/          # Reusable UI components
│   └─ sections/       # Page‑specific sections (PostOpportunity, Saved)
├─ lib/                # Utility modules (axios instance, API helpers)
├─ styles/             # Global CSS, design tokens
└─ package.json        # Project dependencies and scripts
```

## Key Components
- **PostOpportunitySection.tsx** – Form for posting a new opportunity. Handles state, validation, and submission to the backend.
- **SavedSection.tsx** – Displays a list of saved opportunities for the current user.
- **axios.ts** – Configured Axios instance with base URL, interceptors for auth tokens, and error handling.

## Development Setup
1. **Install dependencies**
   ```bash
   npm ci
   ```
2. **Run the development server**
   ```bash
   npm run dev
   ```
   The app will be available at `http://localhost:3000`.
3. **Environment variables** – Create a `.env.local` with:
   ```
   NEXT_PUBLIC_API_URL=https://api.example.com
   ```

## Styling & Design System
- **Colors**: Custom HSL palette with dark background (`#0a0a0a`) and accent cyan (`#00bcd4`).
- **Typography**: Google Font *Inter* – sleek, legible.
- **Glass‑morphism**: Components use semi‑transparent backgrounds (`bg-white/5`) and backdrop blur for a premium feel.
- **Micro‑animations**: Hover transitions on buttons, focus border color transitions.

## API Integration
All API calls go through the Axios instance (`/lib/axios.ts`). Example usage:
```tsx
import axios from '@/lib/axios';
await axios.post('/opportunities', formData);
```
Error handling bubbles up to UI via toast notifications.
